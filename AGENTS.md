# AI Implementation Guidelines

このリポジトリでAIが実装する際の基本方針:

1. 最小で動く変更を優先する。
2. 共通化は「重複が実際に発生している」「再利用価値が明確」の場合のみ行う。
3. あまり意味のない関数分割はしない。
4. 1回しか使わない薄いラッパー関数は作らない。
5. 既存の命名・構成・責務境界（domain / app / adapters / infra）を崩さない。
6. 変更時はテストを追加または更新し、意図した挙動を明示する。
7. 仕様を変える場合は、先に影響範囲（設定・実行・research）を確認する。
8. 迷ったときは、複雑化より可読性と保守性を優先する。

運用メモ:
- まずは局所的に実装し、重複が増えてから共通化する。
- 早すぎる抽象化は避ける。
